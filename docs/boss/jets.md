## 描述

Jets 是一個紙飛機形 Boss，將場地邊牆作為武器使用。它以速度為基礎的連續位移在場地中流暢移動，絕不瞬間傳送。攻擊循環交替進行側牆衝刺（附有可見預告線）、Z 字形掃場（途經四個航點並各自射擊）以及瞄準散射。血量低於一半後，每次主要動作完成時都會觸發散射爆發。

---

## 移動常數

| 常數 | 數值 | 用途 |
|---|---|---|
| `JETS_GLIDE_SPEED` | 200 像素/秒 | 滑行至衝刺起始位置時的速度 |
| `JETS_DASH_SPEED` | 650 像素/秒 | 實際衝刺穿越場地時的速度 |
| `JETS_SWEEP_SPEED` | 280 像素/秒 | Z 字形掃場穿越航點時的速度 |
| `JETS_ARRIVE_DIST` | 8 像素 | 判定「抵達」的距離閾值 |

## 攻擊循環

Jets 持續循環以下六個移動/攻擊階段。

```
階段 0：待機尋敵
  └── 以 ENEMY_SEEK_ACCEL 加速朝玩家移動
  └── 計時結束（JETS_ACTION_PAUSE = 1.2 秒）→ 進入階段 1

階段 1：滑行至衝刺起始牆
  └── glideToward() 以 JETS_GLIDE_SPEED 滑向選定牆邊
  └── 抵達後 → 進入階段 2

階段 2：預告停頓（0.5 秒）
  └── bossTelegraphLines 顯示衝刺路徑線
  └── 計時結束 → 進入階段 3

階段 3：高速衝刺至對側牆
  └── glideToward() 以 JETS_DASH_SPEED 衝向對側牆
  └── 抵達後：射擊一發，激怒狀態觸發散射 → 進入階段 4

階段 4：Z 字形掃場穿越 4 個航點
  └── glideToward() 以 JETS_SWEEP_SPEED 逐一通過 JETS_ZSWEEP_WPS
  └── 每抵達一個航點時射擊一發
  └── 激怒狀態在最後一個航點觸發散射
  └── 全部航點完成後 → 進入階段 5

階段 5：瞄準散射 + 返回待機
  └── 朝玩家發射扇形彈幕（BOSS_FAN_SPREAD = 0.22 弧度）
  └── → 返回階段 0
```

### Z 字形掃場航點（JETS_ZSWEEP_WPS）

```
(40, 50) → (320, 50) → (40, 224) → (320, 224)
```

（左上 → 右上 → 左下 → 右下，形成 Z 字形路徑。）

---

## 激怒閾值

- 觸發條件：**血量 ≤ 50%**（250 血量中的 125 以下）
- **速度倍率：** `maxSpeed` ×1.5（60 → 有效 90 像素/秒）
- **散射爆發**（`JETS_SCATTER_COUNT = 8` 發，速度 `JETS_SCATTER_SPEED = 150 像素/秒`）以放射狀在以下時機觸發：
  - 完成衝刺後（階段 3 → 階段 4）
  - 完成最後一個 Z 字形航點後（階段 4 → 階段 5）

## AI 狀態欄位

| 欄位 | 用途 |
|---|---|
| `bossPhase` | 當前階段（0–5） |
| `bossTimer` | 計時階段的倒數計時 |
| `bossEnraged` | 是否已啟用激怒速度與散射 |
| `bossDashTarget` | 當前滑行目標位置 `{x, y}` |
| `bossWaypointIdx` | Z 字形掃場航點清單的索引（階段 4） |
| `bossTelegraphLines` | 階段 2 預告期間顯示的角度陣列 |

## 備註

- Jets 不參與通用 `updateEnemyAi` 尋敵迴圈，所有移動邏輯由 `bossWeapon.ts` 中的 `updateJetsPattern` 函式處理。
- 階段 2 的預告時間 0.5 秒刻意短於 Orthogon 的 0.8 秒，使 Jets 的戰鬥節奏更為緊湊。
- 初始 `bossTimer = 1.5 秒` 讓玩家在 Jets 首次出現時有短暫的緩衝時間。
- 流暢移動依賴 `glideToward()` 函式，每個畫格根據目標方向設定速度，抵達時回傳 `true`。
